## Problem Statement

E2E tests are failing with a 10-second timeout when waiting for seed data generation to complete. The actual data generation process can take longer than 10 seconds, especially in E2E testing environments where:

1. Database operations may be slower
2. The dev server may have concurrent requests
3. Data generation involves complex statistical calculations

**Evidence from Previous Run**:
- Error: `Timed out 10000ms waiting for expect(locator).toBeVisible()`
- 4/4 E2E tests failed with identical timeout
- Tests waiting for: `getByText('テストデータの生成が完了しました')`

## Root Cause Analysis

1. **Insufficient Timeout**: `playwright/data-flow.spec.ts` lines 34, 64, 102 use `{ timeout: 10000 }` (10 seconds)
2. **No Graceful Degradation**: Tests fail immediately on timeout instead of retrying or providing context
3. **Production-Ready API**: The `/api/seed` endpoint works correctly but needs adequate time for completion

## Proposed Solution

### Option 1: Increase Timeout (Selected) ✅
**Change**: Increase E2E timeout from 10s to 30s across all data generation tests
**Rationale**:
- 30s allows sufficient time for database operations and statistical calculations
- Still fast enough for practical CI/CD pipelines
- Aligns with Playwright's default webServer timeout (120s)
- No changes to production code

**Files to Modify**:
- `playwright/data-flow.spec.ts` (lines 34, 64, 102)

### Option 2: Database Mocking (Deferred)
**Change**: Mock database responses in E2E tests
**Rationale**: Faster tests but less realistic
**Decision**: Deferred - E2E tests should validate real data flow

## Implementation Plan

### Step 1: Update Test Timeouts
```typescript
// Before
await expect(page.getByText('テストデータの生成が完了しました')).toBeVisible({ timeout: 10000 });

// After

```
