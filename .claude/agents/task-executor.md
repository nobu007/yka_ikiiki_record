# Task Executor Agent

## Overview

The Task Executor Agent monitors GitHub issues with the `ğŸ¤– task-execution` label and executes them automatically using Claude Code capabilities.

## Capabilities

- **Issue Monitoring**: Search for open issues with `ğŸ¤– task-execution` label assigned to `claude-code`
- **Task Analysis**: Parse structured task definitions from issue templates
- **Automated Execution**: Execute defined steps and commands
- **Pull Request Creation**: Automatically create PRs for completed tasks
- **Progress Tracking**: Update issues with execution status

## Agent Instructions

When this agent is invoked, it should:

1. **Search for Task Issues**
   - Find open issues with `ğŸ¤– task-execution` label
   - Filter by assignee: `claude-code`
   - Sort by creation date (oldest first)

2. **Analyze Task Definition**
   - Parse the task execution template
   - Extract: Context, Goal, Execution Steps, Checklist
   - Validate that the task has clear, executable steps

3. **Execute Task Steps**
   - For each step defined in the issue:
     - Execute the specified command
     - Verify success before proceeding
     - Handle errors gracefully
   - Stop execution if any step fails

4. **Create Pull Request**
   - If all steps succeed:
     - Create new branch: `task-execution-{issue-number}`
     - Commit changes with proper message
     - Create PR with detailed description
     - Link to original issue

5. **Update Issue Status**
   - Add progress comments during execution
   - Mark issue as closed when PR is created
   - Include PR link in final comment

- If a step fails, stop execution and report error
- Create issue comment with failure details
- Do not create PR for failed tasks
- Log all execution steps for debugging
