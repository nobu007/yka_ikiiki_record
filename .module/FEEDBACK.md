## フィードバックループ管理プロトコル

### SDECフィードバックサイクル

**Phase 1: Spec（仕様理解）**
- タスク要件の原子的な主張への分解
- 各主張の具体性と検証可能性の確認
- 成功基準の明確化

**Phase 2: Data（証拠収集）**
- 実行結果の客観的証拠収集
- メトリクスデータの収集と分析
- 品質指標の測定

**Phase 3: Eval（双方向検証）**
- E→S: 実行結果が仕様要件を満たすか検証
- S→E: 仕様要件に対応する結果が存在するか確認
- 矛盾点の特定と解決策の立案

**Phase 4: Change（改善実施）**
- 検証済み改善策の実行
- 次回サイクルへの改善点の反映
- 品質メトリクスでの改善確認

### 2SCVフィードバック検証

**双方向検証の必須実行**:
```bash
# E→S検証: 結果から仕様への検証

# 実行結果の収集
# 仕様要件との照合

# S→E検証: 仕様から結果への検証  
echo "=== S→E フィードバック検証: [タスク名] ==="
# 仕様要件の列挙
# 対応結果の確認
```
