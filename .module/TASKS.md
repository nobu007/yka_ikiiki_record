## タスク管理プロトコル

### SDECサイクル管理

**Phase 1: Spec（仕様理解）**
- 要求を原子的な主張に分解
- 各主張の具体性と検証可能性を確認
- 成功基準の明確化

**Phase 2: Data（証拠収集）**
- grep/findコマンドによる客観的証拠収集
- 既存コードとドキュメントの分析
- 数値データによる現状把握

**Phase 3: Eval（双方向検証）**
- E→S: 証拠が主張を支持するか検証
- S→E: 主張に対応する証拠が存在するか確認
- 矛盾する証拠の特定と解決

**Phase 4: Change（変更実施）**
- 検証済み変更のみを実行
- 段階的実施と継続的検証
- 品質メトリクスでの完了確認

### 2SCV検証プロトコル

**双方向検証の必須実行**:
```bash
# E→S検証: 証拠から仕様への検証
echo "=== E→S 検証: [タスク名] ==="
# 証拠収集コマンド実行
# 仕様支持の確認

# S→E検証: 仕様から証拠への検証  
echo "=== S→E 検証: [タスク名] ==="
# 仕様要件の列挙
# 対応証拠の確認
```

### ACR自律的補完

**不足情報の補完ルール**:
- ユーザーへの質問は禁止
- 既存コードからの推論を強制
- 類似パターンからの仮説生成
- 仮説も証拠で検証してから使用
