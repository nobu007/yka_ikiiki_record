# SPEC: reloadPage

## 概要
- **モジュール**: `src/lib/constants/browser.ts`
- **責務**: ブラウザで現在のページをリロードする
- **関連する不変条件**: なし

## 入力契約
| パラメータ | 型 | 制約 | デフォルト |
|-----------|-----|------|-----------|
| (なし) | - | 引数なし | - |

## 出力契約
| 戻り値 | 型 | 保証する条件 |
|--------|-----|-------------|
| void | void | 関数は常にundefinedを返す。副作用としてページがリロードされる |

## 動作契約
| 環境 | 動作 | 備考 |
|------|------|------|
| ブラウザ | `window.location.reload()` を呼び出し、ページをリロード | ネットワーク経由で現在のURLを再読み込み |
| Node.js/jsdom | `Error: Not implemented: navigation` がスローされる | jsdomはページ遷移を模拟しない |

## エラー契約
| 条件 | 例外/レスポンス | 備考 |
|------|----------------|------|
| jsdom環境 | Errorがスローされる | テスト環境では必ずモックが必要 |
| ブラウザ環境 | エラーはスローされない | ページがリロードされる |

## 境界値
| 入力 | 期待出力 | 備考 |
|------|---------|------|
| 正常呼び出し（ブラウザ） | ページがリロードされる | 副作用 |
| 連続呼び出し（ブラウザ） | 2回目以降は実行されない（ページがリロード済み） | - |
| jsdom環境で呼び出し | Errorがスローされる | "Not implemented: navigation" |

## テスト要件
- **必須モック**: jsdom環境では `window.location.reload` を vi.fn() でモックすること
- **検証事項**: モックが1回呼び出されることを確認すること
- **エラーテスト**: モックなしで呼び出した場合、Errorがスローされることを確認すること（オプション）

## 使用例
```typescript
// 基本使用
reloadPage();

// エラーハンドラー内での使用
<button onClick={reloadPage}>再試行</button>

// 成功後のリロード
const handleSuccess = async () => {
  await saveData();
  reloadPage();
};
```

## 実装ノート
- `window.location.reload()` はページを完全に再読み込みする
- React Routerなどのクライアントサイドルーティングを使用する場合、この関数の代わりに router.navigate() を使用することを検討
- テスト環境では必ずモックが必要
